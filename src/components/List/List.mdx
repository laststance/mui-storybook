# List

A list display component that arranges items vertically.

## Overview

The List component is for grouping and displaying related items. It supports a wide range of uses, from simple text lists to complex lists containing icons, avatars, and checkboxes.

Key features:
- **Flexible Composition**: Can combine various elements like text, icons, avatars, checkboxes
- **Interactive**: Create clickable items with ListItemButton
- **Nest Support**: Easily implement hierarchical lists
- **Secondary Actions**: Can place additional action buttons on each item
- **Dividers**: Add visual separation with Divider

Main subcomponents:
- **ListItem**: Each item in the list
- **ListItemButton**: Clickable list item
- **ListItemIcon**: Icon display area
- **ListItemAvatar**: Avatar display area
- **ListItemText**: Primary and secondary text
- **ListItemSecondaryAction**: Secondary action placement area

# List

アイテムを縦に配置するリスト表示コンポーネント。

## 概要

Listコンポーネントは関連するアイテムをグループ化して表示するためのものです。シンプルなテキストリストから、アイコン、アバター、チェックボックスを含む複雑なリストまで、幅広い用途をサポートします。

主な機能：
- **柔軟な構成**: テキスト、アイコン、アバター、チェックボックスなど様々な要素を組み合わせられます
- **インタラクティブ**: ListItemButtonでクリック可能なアイテムを作成できます
- **ネストサポート**: 階層構造のリストを簡単に実装できます
- **セカンダリアクション**: 各アイテムに追加のアクションボタンを配置できます
- **区切り線**: Dividerで視覚的な分離を追加できます

主なサブコンポーネント：
- **ListItem**: リストの各アイテム
- **ListItemButton**: クリック可能なリストアイテム
- **ListItemIcon**: アイコン表示領域
- **ListItemAvatar**: アバター表示領域
- **ListItemText**: プライマリおよびセカンダリテキスト
- **ListItemSecondaryAction**: セカンダリアクション配置領域

## Usage Example

```tsx
import List from '@mui/material/List'
import ListItem from '@mui/material/ListItem'
import ListItemButton from '@mui/material/ListItemButton'
import ListItemIcon from '@mui/material/ListItemIcon'
import ListItemText from '@mui/material/ListItemText'
import ListItemAvatar from '@mui/material/ListItemAvatar'
import Avatar from '@mui/material/Avatar'
import InboxIcon from '@mui/icons-material/Inbox'

// Basic list
<List>
  <ListItem>
    <ListItemText primary="Item 1" />
  </ListItem>
  <ListItem>
    <ListItemText primary="Item 2" />
  </ListItem>
</List>

// List with icons
<List>
  <ListItem disablePadding>
    <ListItemButton>
      <ListItemIcon>
        <InboxIcon />
      </ListItemIcon>
      <ListItemText primary="Inbox" />
    </ListItemButton>
  </ListItem>
</List>

// List with avatars
<List>
  <ListItem>
    <ListItemAvatar>
      <Avatar>
        <FolderIcon />
      </Avatar>
    </ListItemAvatar>
    <ListItemText primary="Photos" secondary="Jan 9, 2024" />
  </ListItem>
</List>

// Nested list
<List>
  <ListItemButton onClick={handleClick}>
    <ListItemIcon>
      <InboxIcon />
    </ListItemIcon>
    <ListItemText primary="Inbox" />
    {open ? <ExpandLess /> : <ExpandMore />}
  </ListItemButton>
  <Collapse in={open}>
    <List component="div" disablePadding>
      <ListItemButton sx={{ pl: 4 }}>
        <ListItemText primary="Starred" />
      </ListItemButton>
    </List>
  </Collapse>
</List>

// With secondary action
<List>
  <ListItem
    secondaryAction={
      <IconButton edge="end" aria-label="delete">
        <DeleteIcon />
      </IconButton>
    }
  >
    <ListItemText primary="Item" />
  </ListItem>
</List>
```

## Accessibility

- Appropriate `role` attributes are automatically set for list items
- Interactive elements (ListItemButton) support keyboard operation
- Always specify `aria-label` for icon-only buttons
- Use primary and secondary text hierarchy appropriately
- For lists with checkboxes, make the selection state clearly visible
- Express nested list structure with appropriate ARIA attributes
</En>

## アクセシビリティ

- リストアイテムには適切な`role`属性が自動的に設定されます
- インタラクティブな要素（ListItemButton）はキーボード操作をサポートします
- アイコンのみのボタンには必ず`aria-label`を指定してください
- プライマリテキストとセカンダリテキストの階層を適切に使用してください
- チェックボックス付きリストでは、選択状態を明確に視覚化してください
- ネストされたリスト構造を適切なARIA属性で表現してください
