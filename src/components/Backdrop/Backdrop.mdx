## Overview

The Backdrop component displays a semi-transparent overlay over the entire screen or specific elements. Used for displaying loading states, modal dialog backgrounds, or directing user attention to specific elements. Typically used in combination with CircularProgress or messages.

Key features:
- Customizable transparency and color
- z-index control
- Transition effects
- Click event handling
- Combination with loading indicators

</En>
<Ja>

## 概要

Backdropコンポーネントは、画面全体または特定の要素の上に半透明のオーバーレイを表示します。ローディング状態の表示、モーダルダイアログの背景、または特定の要素にユーザーの注意を向けるために使用されます。通常、CircularProgressやメッセージと組み合わせて使用されます。

主な機能:
- 透明度と色のカスタマイズ
- z-indexの制御
- トランジション効果
- クリックイベントの処理
- ローディングインジケーターとの組み合わせ

</Ja>

## 使用例

</Ja>

```tsx
import Backdrop from '@mui/material/Backdrop'
import CircularProgress from '@mui/material/CircularProgress'
import Button from '@mui/material/Button'
import React from 'react'

function MyBackdrop() {
  const [open, setOpen] = React.useState(false)

  const handleClose = () => {
    setOpen(false)
  }

  const handleOpen = () => {
    setOpen(true)
  }

  return (
    <div>
      <Button variant="contained" onClick={handleOpen}>
        Show loading
      </Button>

      <Backdrop
        sx={{
          color: '#fff',
          zIndex: (theme) => theme.zIndex.drawer + 1
        }}
        open={open}
        onClick={handleClose}
      >
        <CircularProgress color="inherit" />
      </Backdrop>
    </div>
  )
}

// Backdrop with custom content
function CustomBackdrop() {
  const [open, setOpen] = React.useState(false)

  return (
    <Backdrop
      sx={{ color: '#fff', zIndex: 1300 }}
      open={open}
      onClick={() => setOpen(false)}
    >
      <Box sx={{ textAlign: 'center' }}>
        <CircularProgress color="inherit" />
        <Typography sx={{ mt: 2 }}>Loading...</Typography>
      </Box>
    </Backdrop>
  )
}
```

<En>

## Accessibility

- While Backdrop is open, focus should be managed appropriately
- When indicating loading state, consider using the `aria-busy` attribute
- For important messages, consider including an element with `role="alert"`
- For keyboard users, consider implementing functionality to close with the Escape key

</En>
<Ja>

## アクセシビリティ

- Backdropが開いている間、フォーカスは適切に管理される必要があります
- ローディング状態を示す場合、`aria-busy`属性の使用を検討してください
- 重要なメッセージの場合、`role="alert"`を持つ要素を含めることを検討してください
- キーボードユーザーのために、Escapeキーで閉じる機能の実装を検討してください

</Ja>
